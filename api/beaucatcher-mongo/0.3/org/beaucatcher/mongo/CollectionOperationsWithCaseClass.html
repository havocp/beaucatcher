  26. def syncDAO [E] (implicit chooser: CollectionOperationsWithCaseClass.this.SyncDAOChooser[E, _]): org.beaucatcher.mongo.SyncDAO[org.beaucatcher.bson.BObject, E, IdType, _]

    This lets you write a function that generically works for either the entity (often case class) or BObject results.

    This lets you write a function that generically works for either the entity (often case class) or BObject results. So for example you can implement query logic that supports both kinds of result.

       syncDAO[BObject].find() // returns BObject results
   syncDAO[MyCaseClass].find() // returns MyCaseClass results
   def myQuery[E] = syncDAO[E].find(... query ...) // generic query

    With methods such as distinct(), you probably need the syncDAO[E,V] flavor that lets you specify the type of field values. With methods that don't return objects, such as count(), you can use the syncDAO flavor with no type parameters.
