org.beaucatcher.mongo

AsyncCollection

trait AsyncCollection extends ReadOnlyAsyncCollection with Collection

Trait containing all read and write operations on a collection, in asynchronous form, with wire encoding and decoding delegated to implicit typeclass parameters. For the synchronous equivalent, see SyncCollection. To avoid the use of implicits, see BoundAsyncCollection.

The recommended way to obtain an instance of AsyncCollection is from the async property on CollectionAccess, which would in turn be implemented by the companion object of a case class representing an object in a collection. For example you might have case class Foo representing objects in the foo collection, with a companion object object Foo which implements CollectionAccess. You would then write code such as:

   Foo.async[BObject].find() // obtain results as a BObject
   Foo.async[Foo].find()     // obtain results as a case class instance
   Foo.async.count()         // entity type not relevant

This is only a convention though, of course there's more than one way to do it.
