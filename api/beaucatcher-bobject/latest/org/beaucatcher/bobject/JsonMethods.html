JsonMethods

trait JsonMethods [SchemaType <: Product] extends AnyRef

The idea of this trait is to implement a backbone.js-style set of methods for a MongoDB collection. backbone.js just seems like a good "spec" for a set of CRUD methods in REST format. You can pretty easily adapt this to map to HTTP differently, though.

See http://documentcloud.github.com/backbone/#Sync

My still-untested interpretation follows.

CRUD maps to REST like this: create = POST /collection read = GET /collection[/id] update = PUT /collection/id delete = DELETE /collection/id

Create and Update should return the modified fields (the fields different from those passed in) though they are allowed to return unmodified fields, with an efficiency cost.

Read should always return all fields of the object. Without the /id, it returns a JSON array of all objects.

Delete I guess should return an empty body, though I'm not sure what backbone.js expects.

FIXME: This class probably needs some way to pass in application-specific data (such as the currently-logged-in-user) to the various methods. I guess you could use thread local but maybe there's something cleaner.

  20. def modifyIncomingJson (path: Option[String], jvalue: JValue): JObject

    Override this if you want to modify JSON that's coming in.

    Override this if you want to modify JSON that's coming in. You can also throw a JsonValidationException from here. This method will be called before validating the JSON against the case class schema. This method must return an object, not any other kind of JValue.

    By default this method forces the JValue to be a JObject and adds an "_id" field if there wasn't one before.     In this method, you could also "fix up" JSON (for example add default values for missing fields) before the JSON gets validated.

    This method is called for creating and for updating.
